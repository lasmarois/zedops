## Preview Panel Behavior

### Collapsible Preview Panels (Overview Tab)

**Collapsed (Default):**
- Shows last 5-10 lines of logs
- Shows last 2-3 RCON commands
- Basic controls (pause, clear)
- [â–¼ Expand View] button

**Expanded (Click Expand):**
- Opens full-screen modal OR
- Navigates to dedicated tab
- User preference: Modal vs Tab navigation

**Implementation Options:**

**Option A: Modal Overlay**
```tsx
<Dialog open={logExpanded} onOpenChange={setLogExpanded}>
  <DialogContent className="max-w-6xl h-[80vh]">
    <DialogHeader>
      <DialogTitle>Server Logs: jeanguy</DialogTitle>
    </DialogHeader>
    {/* Full log viewer component */}
  </DialogContent>
</Dialog>
```

**Option B: Navigate to Tab**
```tsx
const expandLogs = () => {
  setActiveTab('logs');  // Switch to Logs tab
};
```

**Recommendation:** Option B (Navigate to Tab)
- Cleaner UX (no nested scrolling)
- Logs/RCON already have dedicated tabs
- Preview is for quick glance only
- Full functionality available in tab

## Responsive Behavior

### Mobile (< 768px)

**Sidebar:**
- Collapses to hamburger menu
- Overlay drawer from left
- Push content or overlay?
  - **Recommendation:** Overlay (keeps content visible)

**Tables:**
- Convert to stacked cards
- Each row becomes a card
- Priority info shown first
- Actions in card footer

**Dashboard Stats:**
- Stack vertically (1 column)
- Full width cards

**Preview Panels:**
- Show collapsed by default
- Expand to full screen modal (no space for inline)

### Tablet (768px - 1024px)

**Sidebar:**
- Collapsed by default (icon-only)
- Expand on hover or click
- Labels hidden, icons shown

**Tables:**
- 2 columns on wider tablets
- Some columns hidden (show on click)

**Dashboard Stats:**
- 2 columns

### Desktop (> 1024px)

**Sidebar:**
- Full width (240px)
- Always visible
- Sections expanded

**All Components:**
- Full layout as designed
- All columns visible
- Maximum information density
